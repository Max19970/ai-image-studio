# Image API parameters covered by the UI

Generation parameters are owned by logical modules in:

```txt
src/entities/generation-params/fields/<param>/
  param.ts        # copy/options/state/capability/snapshot/restore/payload metadata
  definition.ts  # UI field definition used by composer parameter placements
```

`src/entities/generation-params/logicalRegistry.ts` is the single entry point for parameter behavior. It builds OpenAI-compatible payload fragments, captures request snapshots, sanitizes persisted snapshot params, restores composer state from history, and normalizes persisted composer params.

## Generate endpoint

`POST /v1/images/generations`, JSON body:

- `model` — controlled by `include-model`.
- `prompt` — base request field, not a parameter module.
- `background` — `background` module.
- `moderation` — `moderation` module.
- `n` — `n` module.
- `output_compression` — `output-compression` module; sent only when the selected `output_format` is not the default `png`.
- `output_format` — `output-format` module; omitted for default `png` compatibility.
- `partial_images` — `partial-images` module; sent only when streaming is enabled.
- `quality` — `quality` module.
- `response_format` — `response-format` module.
- `size` — `size` module.
- `stream` — `stream` module.
- `style` — `style` module.
- `user` — `user` module.

## Edit endpoint

`POST /v1/images/edits`, multipart/form-data:

- `image` files — target/reference attachments, not parameter modules.
- `mask` file — mask attachment, not a parameter module.
- `model` — controlled by `include-model`.
- `prompt` — base request field, not a parameter module.
- `background` — `background` module.
- `input_fidelity` — `input-fidelity` module; edit mode only.
- `n` — `n` module.
- `output_compression` — `output-compression` module.
- `output_format` — `output-format` module.
- `partial_images` — `partial-images` module.
- `quality` — `quality` module.
- `response_format` — `response-format` module.
- `size` — `size` module.
- `stream` — `stream` module.
- `style` — `style` module when supported by the provider.
- `user` — `user` module.

## Raw JSON overrides

Any parameter that is not exposed by a form field can be added through `Raw JSON overrides`.

Raw JSON is parsed by the `raw-json` module and merged after all form-owned payload fragments. This means values from raw JSON intentionally have priority over values generated by UI controls.

## Snapshots and restore

Request snapshots store module-owned parameter values through `captureGenerationRequestParamsSnapshot(...)`. Restoring a historical request uses `restoreImageParamsFromRequestSnapshot(...)`, which restores values and include toggles by looking at module `snapshotKeys`, `includeKey`, and `payloadKeys` metadata.
